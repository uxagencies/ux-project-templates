# Design Brief
### [Project name]

> [!NOTE]
> **Why this document exists.** The brief is the contract for *what good looks like* — written after a project is greenlit and before design starts, ideally by client and agency together. A shared brief is the cheapest insurance against the most expensive problem in design work: discovering in the final review that you were building different things.

> [!IMPORTANT]
> **Brief vs. RFP.** If you're still choosing an agency, you want an [RFP](rfp-template.md), not this. This assumes you've picked a partner and now need everyone aligned.

## The one-liner

> [!TIP]
> If you can't say what this project is in one sentence, the team can't either. Write it last if you have to — but write it.

> *In one sentence: we are **[doing what]** for **[whom]** so that **[outcome]**.*
